##前言

######今年九月底从上一家公司离职开始就没怎么使用Android Studio了，前几日见团队成员在用Android Studio进行JNI接口测试，不由得想起了使用Android Studio的日子。趁着还有点印象，写一写我开发时常用的快捷键吧。

  • Alt + Enter :

    最常用的功能，没有之一，主要用来补全代码，自动导包等等

  • Ctrl + Alt + L ：

    格式化代码，其实就是代码对齐

  • Ctrl + Alt + S

    设置

  • 删除光标所在一行

    Ctrl + Y

  • 复制光标所在一行

    Ctrl + D

  • Shift + Alt + ↑或者Shift + Alt + ↓

    上下移动光标所在行，选中多行后也可移动

  • Shift + F6

    变量重命名

  • Ctrl + Alt + F

    局部变量转换为全局变量，这边值得多说一点，当我们不知道所调用方法返回什么时候，我们可以使用Alt + Enter进行代码补齐（自动添加局部变量），然后Shift + F6进行变量重命名，最后调用Ctrl + Alt + F把局部变量转换为全局变量。（特别适合在网络数据请求回来过后使用，不用鼠标会大大提高开发效率）

  • Ctrl + Alt + →或者Ctrl + Alt + ←

    返回/前进到上一次编辑位置

  • Ctrl + N

    查找类

  • Ctrl + F

    本类中查找

  • Ctrl + Shift + F

    查找

  • Ctrl + Shift + R

    查找并替换

  • Shift + Shfit(连续按两次Shift)

    全局查找

  • Tab

    ######假如我们手滑点了setTextColor()方法，但是我们真正需要的是setText方法，删掉重写会异常麻烦，把光标移动到set后，重新打T，就会出现setText…等等一系列选项，选中你需要的，按一下tab键，自动帮你把选中的留下来，上一个错误的会自动废弃掉而不用手动删除。（第三方SDK具有很多又长又很相近的方法名，这个办法绝对是开发利器）


  • /** + Enter

    代码注释快捷键，自动添加方法里面的参数param，最好写的清楚点，方便其他人调用时查看

  • Ctrl + Q 或者 Ctrl + P

    可以查看到方法里面的参数或者代码注释，而不用点进方法中进行详细查看(截图接上图)

  • xxx.sout

    System.out.println(xxx)

  • Logi

    Log.i(TAG,String),同理Logv Logd Logw Loge 等等

  • psf 和 psfs

    public static final 和 public static final String

  • const

    public static final int i = 随机一个int数

  • xxx.for

    xxx的循环,如下图：共三种两类循环方式，增强for循环和普通for循环，第三种是倒序的普通for循环

  • xxx.switch

    根据xxx进行switch case选择;截图中比较有意思的就是有居然sout选项
    看到这里

    ######聪明的同学应该已经想到xxx.if 或者 xxx.while了
    ######更聪明的同学应该已经想到xxx.switch 和 if else可以通过 Alt + Enter进行转换了（仅供玩耍，没实际作用）
    ######更更聪明的同学应该已经去试验Android Studio能不能识别三元运算符等等…更高级的写法了（开发别这样写，看不明白，装逼可以）

  • 比较有意思的玩法

    因为Android Studio是基于IDE进行开发的，所以有很多你意想不到的快捷键，当然也包括xml文件的快捷键，但不经常用到，具体可以参照Ctrl + Alt + S → Live Templates
